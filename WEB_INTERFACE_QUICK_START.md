# Web Interface Quick Start Guide

This project includes multiple web-based interfaces that provide a simpler way to interact with the system without complex panels.

## Available Web Interfaces

### 1. Main Dashboard (`dashboard.html`)
- **Purpose**: Real-time monitoring and control of the autonomous Elasticsearch system
- **Features**: 
  - Cluster metrics visualization
  - Agent status monitoring
  - Optimization controls
  - Research insights display
  - Chat/command interface

### 2. Enhanced AI Environment Dashboard (`enhanced-ai-environment-dashboard.html`)
- **Purpose**: Advanced interface for the 48-layer memory system
- **Features**:
  - 48-layer memory visualization
  - Multi-agent coordination controls
  - Advanced analytics

### 3. Collaboration Hub (`collab-hub.html`)
- **Purpose**: Real-time communication between multi-platform agents
- **Features**:
  - WebSocket-based messaging
  - Platform-specific channels
  - Real-time updates

## How to Start the Web Interfaces

### Option 1: Simple Start (Recommended)
Run the main system which includes the dashboard:

```bash
cd c:\autonomous-elasticsearch-evolution-agent
node start-ai-environment.js
```

Then open your browser and go to: `http://localhost:4001`

### Option 2: Start Dashboard Server Only
```bash
cd c:\autonomous-elasticsearch-evolution-agent
node dashboard-server.js
```

Then open your browser and go to: `http://localhost:3000`

### Option 3: Start Enhanced AI Environment
```bash
cd c:\autonomous-elasticsearch-evolution-agent
node ai-environment-api.js
```

Then open your browser and go to: `http://localhost:4001`

## Using the Dashboard Interface

Once you open the dashboard in your browser, you'll see:

1. **Orchestration Status** - Shows the current state of the system
2. **Cluster Metrics** - Displays health, memory usage, fragmentation, and latency
3. **System Status** - Shows the status of each agent (optimizer, research, coding, etc.)
4. **Research Insights** - Displays insights generated by the research agent
5. **Recent Optimizations** - Shows optimizations applied to the system

### Controls:
- **Run Optimization Cycle** - Starts the optimization process
- **Analyze Cluster Now** - Forces an immediate cluster analysis
- **Run Simulations** - Executes simulation engine
- **Federation Sync** - Synchronizes agent states
- **Reset Agent State** - Resets the agent state

## Troubleshooting

### If the dashboard doesn't load:
1. Make sure the server is running (`node start-ai-environment.js`)
2. Check that you're accessing the correct URL
3. Look at the console for any error messages

### If you see "Disconnected" status:
1. Verify the server is running
2. Check your firewall settings
3. Make sure no other process is using the required ports (3000, 4001)

## Port Reference

- **Port 3000**: Main dashboard server
- **Port 4000**: Collaboration hub server
- **Port 4001**: Enhanced AI environment dashboard
- **Port 3001**: Local agent
- **Port 3002**: Background agent
- **Port 3003**: Cloud agent

## Quick Commands

To start the system with all features:

```bash
# Start everything
node start-all-services.js

# Or just the AI environment
node start-ai-environment.js

# Or just the dashboard
node dashboard-server.js
```

The web interface provides a much simpler way to interact with the system compared to complex panels, giving you visual feedback and easy controls.
